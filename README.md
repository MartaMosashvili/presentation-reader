# პრეზენტაციის წამკითხველი (Presentation Reader)

ვებ აპლიკაცია, რომელიც PowerPoint პრეზენტაციას ხმით, ქართულად კითხულობს და სლაიდებს ავტომატურად ცვლის — ტექსტის დასრულებიდან ზუსტად 2 წამში.

**Live demo:** https://presentation-reader.onrender.com
*(Render-ის უფასო სერვისი უმოქმედობისას "იძინებს" — პირველი გახსნა ~1 წუთს შეიძლება მოითხოვდეს.)*

## სწრაფი ტესტირება (2 წუთი)

1. გახსენით [Live demo](https://presentation-reader.onrender.com) *(პირველი ჩატვირთვა ~1 წუთი — უფასო სერვერი "იღვიძებს")*
2. ჩამოტვირთეთ სატესტო ფაილები ამ რეპოდან: [`demo/demo-presentation.pptx`](demo/demo-presentation.pptx) და [`demo/demo-narration.docx`](demo/demo-narration.docx)
3. ატვირთეთ ორივე → აირჩიეთ ხმა (ეკა/გიორგი) → **ხმის გენერაცია** (~1 წუთი) → **პრეზენტაციის დაწყება**

შეუსაბამობის დამუშავების სანახავად ატვირთეთ ნებისმიერი სხვა docx — აპი ზუსტად გეტყვით, რამდენი აბზაცია და რამდენი სლაიდი.

## როგორ მუშაობს

1. **ატვირთვა** — მომხმარებელი ტვირთავს პრეზენტაციას (`.pptx`) და წასაკითხ ტექსტს (`.docx`). წესი: დოკუმენტში **თითო აბზაცი = თითო სლაიდის ტექსტი**, თანმიმდევრობით.
2. **კონვერტაცია** — სერვერი LibreOffice-ით (headless) გარდაქმნის pptx-ს PDF-ად; ბრაუზერი სლაიდებს PDF.js-ით ხატავს, ამიტომ ვიზუალი ორიგინალის იდენტურია.
3. **შესაბამისობის გადამოწმება** — აპი გვერდიგვერდ აჩვენებს თითო სლაიდს და მის ტექსტს; თუ სლაიდებისა და აბზაცების რაოდენობა არ ემთხვევა, მომხმარებელი იღებს ზუსტ გაფრთხილებას და გენერაცია იბლოკება. ბუნდოვან მონაცემებზე აპი ვარაუდით არ მუშაობს.
4. **ხმის გენერაცია** — თითო აბზაცი ცალკე mp3 ფაილად გენერირდება Microsoft Azure-ის **ნატიური ქართული ნეირონული ხმებით** (ეკა / გიორგი — არჩევა ინტერფეისიდან). მთელი აუდიო წინასწარ იქმნება, ამიტომ დაკვრისას TTS სერვისზე დამოკიდებულება აღარ არსებობს.
5. **დაკვრა** — ჩაშენებული ფრეიმი, აუდიო, ავტო-გადართვა დასრულებიდან 2 წამში; პაუზა/გაგრძელება (პაუზა 2-წამიან ინტერვალსაც აჩერებს), წინა/შემდეგი, მინიატურების პანელი ნებისმიერ სლაიდზე გადასახტომად, კითხვის სიჩქარის რეგულირება. დაწყებამდე მთელი აუდიო ბრაუზერში ჩამოიტვირთება — ინტერნეტის გათიშვა დაწყებულ პრეზენტაციას ვერ აჩერებს.

## ტექნოლოგიები

- **Backend:** Node.js / Express, multer (ატვირთვა), mammoth (docx-ის პარსინგი), LibreOffice headless (pptx→PDF)
- **TTS:** Microsoft Azure Speech — `ka-GE-EkaNeural`, `ka-GE-GiorgiNeural` (შედარებული იყო OpenAI TTS-თან და ElevenLabs-თან; Azure ერთადერთია ნატიური ქართული ხმებით)
- **Frontend:** ვანილა JavaScript, PDF.js (ლოკალურად მიწოდებული — CDN-ზე დამოკიდებულების გარეშე)
- **Deploy:** Docker (Render), LibreOffice კონტეინერშივე

## ლოკალურად გაშვება

წინაპირობები: Node.js 18+, LibreOffice (soffice სტანდარტულ გზაზე ან `SOFFICE_PATH` ცვლადში).

```bash
npm install
# შექმენით .env ფაილი (ნიმუში: .env.example):
#   TTS_PROVIDER=azure
#   AZURE_SPEECH_KEY=...
#   AZURE_SPEECH_REGION=northeurope
npm start
# http://localhost:3000
```

## საიმედოობის გადაწყვეტილებები

- ხმა და კონვერტაცია მუშავდება **ატვირთვისას და არა დაკვრისას** — ლაივ პრეზენტაცია გარე სერვისებზე არ არის დამოკიდებული.
- TTS-ის დროებით შეფერხებაზე — ავტომატური განმეორება მზარდი ინტერვალით; ჩავარდნისას პროგრესი ინახება (მაგ. „5/8 მზადაა").
- აუდიოს ჩატვირთვის შეცდომა დაკვრას არ აჩერებს — აპი აცნობებს და გრძელდება.
- ერთდროული კონვერტაციები დაბლოკილია (რესურსების დაცვა), ორმაგი გენერაციის მოთხოვნა უგულებელყოფილია.
- ფაილის ტიპი მოწმდება სერვერზე (და არა მხოლოდ ბრაუზერში), გზები იზოლირებულია path traversal-ისგან.
